# SKILL · 文案规范

> 配套 SPEC：`specs/00-foundation/content-guidelines.spec.md`。与 i18n 协同（约束 key 默认值）。

## 五原则
准确清晰 · 简洁 · 一致（术语/标点/大小写）· 友好克制 · 数字单位走 Intl。

## 速查
| 场景 | 怎么写 |
|---|---|
| 按钮 | 动词/动宾：保存、删除、新建；危险操作明确「永久删除」|
| 确认弹窗 | 标题陈述动作，正文说后果，按钮回显动作（删除/取消，非确定/取消）|
| 空状态 | 说明为何空 + 下一步动作 |
| 错误 | 原因 + 解决建议，不写「出错了」「未知错误」|
| 占位符 | 提示输入内容，不替代 label |
| 标点 | 中文全角、标题不加句号；英文半角、句首大写 |

## 落地
- 默认文案集中在 `@chenzy-design/locale` 语言包，统一审校。
- 同一概念全库术语必须一致。
- 危险/不可逆操作文案在组件 SPEC §8 单列并复核。

## 检查清单
- [ ] 无指责性/无效错误文案。
- [ ] 术语、标点、大小写一致。
- [ ] 数字/日期走 Intl，不写死。
- [ ] 文档示例文案同样合规。
