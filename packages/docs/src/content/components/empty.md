## 使用场景

Empty 用于在「无数据/无搜索结果/无权限/网络错误」等场景下，向用户展示一个友好的占位状态，避免出现空白区域让用户产生页面卡住或加载失败的误解。通过图示、说明文案与可选操作按钮的组合，引导用户下一步操作。

典型使用场景：表格/列表无数据时的占位、搜索结果为空、用户尚未创建任何内容（引导首次使用）、网络错误或无权限的友好提示。支持自定义图示与操作区，适应不同业务含义的空状态。

## 何时使用

在数据区域确实为空、需要向用户给出明确反馈并引导下一步操作时使用 Empty。不适合用于加载中的过渡态（请使用 Skeleton 或 Spin），也不适合用于错误态详情页（应配合专门的错误页面布局）。

## 无障碍

- 根容器设 `role="status"` + `aria-live="polite"`，使屏幕阅读器在空状态动态出现时自动播报。
- 通过 `aria-labelledby` 和 `aria-describedby`（由 `useId` 生成）关联标题与描述文案，使空状态具备完整可访问名称。
- 装饰性插图设 `aria-hidden="true"`，语义完全由文案承载，不依赖图形传递信息。
- Empty 本身不可聚焦；内部操作按钮遵循各自原生 Tab 顺序，满足键盘可达性。

## 文案规范

- **标题简短**（不超过 12 字）陈述事实、不带情绪，描述补充原因或动作建议（每行不超过 24 字）。
- **区分「无数据」与「无搜索结果」**：前者是「还没有」，后者是「没匹配到」，引导动作不同。
- **动作文案用动宾短语**、首词为动词，如「创建项目」「清空筛选」，而非「点击这里」。
- 不用「哎呀 / 出错啦」等卖萌或推责口吻，不向终端用户暴露 error code、stack、字段名。

| ✅ 推荐用法 | ❌ 不推荐用法 |
| --- | --- |
| 暂无数据 · 创建第一条记录后将在此显示 · 创建 | 哎呀，这里什么都没有 |
| 没有找到相关结果 · 试试更换关键词或清空筛选条件 · 清空筛选 | 没有数据 |
| 暂无访问权限 · 如需访问，请联系管理员开通 · 联系管理员 | Error 403：forbidden |

- 区分「用户能解决」（无结果改条件）与「用户不能解决」（无权限联系管理员），不给用户无法执行的建议。
