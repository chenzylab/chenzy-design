## 使用场景

Descriptions（描述列表）用于成组地展示只读的「字段—值」键值对，常见于详情页的基本信息面板、订单信息展示、用户档案等场景。它以规整的标签—值布局呈现一组结构化数据，强调信息的可读性与对齐感。

Descriptions 支持水平与垂直两种布局方向，支持多列分栏展示，可通过 `column` 控制每行显示的字段数。标题与边框均可选，适应从极简到带视觉边界的多种页面风格。

## 何时使用

在需要以结构化方式展示一组只读键值信息时使用 Descriptions。不适合用于可编辑表单（请使用 Form 组件），也不适合用于需要对数据进行操作的场景。

## 无障碍

- 使用 `<dl>/<dt>/<dd>` 原生语义结构，屏幕阅读器可自然感知字段与值的对应关系。
- 标题区域通过 `useId` 生成 id 并以 `aria-labelledby` 关联容器，使整个描述列表具备可访问名称。
- 标签后的冒号通过 CSS `::after` 伪元素实现，不进入无障碍树，避免重复朗读。
- Descriptions 为纯展示组件，无键盘交互需求，不设置 `tabindex`，不进入 Tab 序列。
